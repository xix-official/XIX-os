
# Contributing to XIX ❤️👍

Hello! Thank you for contributing to XIX.
This project has a lot of potential and we want to keep it safe, stable and professional.
Please follow these rules before submitting a Pull Request (PR).

---

## 1. General rules

- Don't send big PRs without explanation. 
If the change is big, open a **discussion** first.
- Do not send PRs with backdoors, viruses or any code that compromises user security.
- The code must **compile correctly**.
- You must deliver **the source code**, not just a ready-made ISO.
- Do not send destructive files that remove important functions. 
If something is experimental, **you must state this clearly**.
- The PR must be compatible with existing architectures or add new support.
- The PR must work with supported hardware or add new support.

---

## 2. What you can send

Contributions are accepted such as:

- Drivers
- Optimizations
- Performance improvements
- Documentation
- Bug fixes
- Installer improvements
- GUI improvements
- New open-source apps
- New commands or tools

---

## 3. What is NOT accepted

- Code without clear license.
- Code copied from other projects without permission.
- PRs that remove essential functions.
- PRs that break the build.
- PRs that change the style of the project without consensus.
- PRs that add unnecessary dependencies.
- Binaries included in the PR.
- Obfuscated code.
- Destructive scripts.
- Hidden telemetry.
- Rear doors.
- Malware.

---

## 4. Code style

To keep the project organized:

- The code must have comments (`#` or `//`).
- Functions must have clear names.
- You must respect the project folder structure.
- Commits must be descriptive and clear.

Examples of good commits:
- `fix: corrected error in the installer`
- `feat: added support for new ARM driver`
- `perf: kernel optimization on AMD64`

---

## ⚠️ IMPORTANT ⚠️

**You must deliver the source code.
ISOs without the corresponding code are not accepted.**
